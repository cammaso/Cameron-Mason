    Nmap scan types & handshake behavior
    FlagScan TypeBehavior
    -sSSYN / StealthSends SYN, gets SYN-ACK, sends RST. Never completes handshake. Less likely to be logged.
    -sTTCP ConnectCompletes full 3-way handshake. Logged by OS. Used when raw sockets unavailable.
    -sUUDPNo handshake. Sends UDP probes; closed ports return ICMP unreachable.
    -sFFINSends FIN only. Closed ports reply RST; open ports ignore. Evades some stateless firewalls.
    -sXXmasSets FIN+PSH+URG. Lights up the header "like a Christmas tree." Same logic as FIN scan.
    -sNNullSends no flags. Closed ports respond RST; open ports ignore.
    -sAACKMaps firewall rules — tells you which ports are filtered vs unfiltered.
    Exam tip
    SYN scan (-sS) is the default Nmap scan and most commonly tested. Know why it's "stealthy" — it never completes the handshake so many older IDS systems don't log it. A SYN flood DoS attack exploits this same mechanism at massive scale.

        Module 4 — Footprinting & Reconnaissance

        Footprinting & Recon

        OSINT techniques, recon tools, DNS enumeration, and passive vs active reconnaissance.

        Passive vs Active Recon
        Passive recon
        Gathering information without directly interacting with the target. Target never knows you're looking. Uses public records, OSINT, search engines, WHOIS, DNS lookups, social media.
        Google Dorks WHOIS Shodan LinkedIn Maltego theHarvester
        Active recon
        Directly interacting with target systems to gather data. Target may detect or log the activity. Includes port scanning, OS fingerprinting, ping sweeps, DNS zone transfers.
        Nmap dig / nslookup Netcraft Traceroute fierce dnsrecon

        Exam tip — Module 4
        Zone transfer (AXFR) is the biggest DNS misconfiguration — if the nameserver allows it, you get every DNS record for the domain. Passive footprinting using Google Dorks (site:, filetype:, inurl:) is also heavily tested. Know: passive = no direct target contact, active = direct contact.

        Module 4 — Social Engineering

        Social Engineering

        Attack types, psychological principles, phishing techniques, and defenses. Lab 4-2 content.

        Social engineering attack types
        Psychological principles exploited
        Authority
        Impersonating IT, management, or law enforcement to make targets comply without questioning.
        Urgency / Scarcity
        "Act now or your account will be suspended." Bypasses critical thinking by creating time pressure.
        Social proof
        "Your colleagues have already updated their passwords." Normalizes the requested action.
        Reciprocity
        Offering a small favor or gift first, then making a request. Creates obligation to comply.
        Liking / Trust
        Building rapport before the ask. Attackers research targets on LinkedIn/social media first.
        Fear
        "Your computer has a virus, call us now." Drives irrational decisions to eliminate perceived threat.
        Phishing subtypes
        Defenses against social engineering
        • Security awareness training — the only reliable defense against human manipulation
        • Verify caller/sender identity via callback using a known, official number — not one provided by the caller
        • Strict procedures for password resets, data sharing, and physical access — never bypass for "urgency"
        • Email filtering, DMARC/SPF/DKIM to reduce spoofed phishing emails
        • Multi-factor authentication (MFA) — limits damage even if credentials are stolen
        • Shred sensitive documents — dumpster diving is a real physical recon technique
        • Tailgating controls — badge-only access, no piggybacking
        Exam tip — Module 4
        Know all phishing subtypes: phishing (mass), spear phishing (targeted), whaling (executives), vishing (voice), smishing (SMS), pharming (DNS redirect). Social engineering succeeds because it targets humans, not technology — the weakest link in any security chain.
        Module 5 — Port Scanning

        Nmap Quick Reference

        Every Nmap flag and technique you need for the Chapter 5 lab and quiz.

        Scan types
        SwitchNameDescription
        -sSSYN / Stealth scanDefault. Half-open scan — sends SYN, resets after SYN-ACK. Stealthy, fast, requires root.
        -sTTCP connect scanFull 3-way handshake. Logged. Used when -sS unavailable (no root).
        -sUUDP scanScans UDP ports. Slow. Important for DNS (53), SNMP (161), DHCP (67).
        -sFFIN scanSends FIN. Open ports ignore, closed send RST. Evades some firewalls.
        -sXXmas scanFIN+PSH+URG. Same logic as FIN scan.
        -sNNull scanNo flags. Open ignore, closed RST.
        -sAACK scanMaps firewall rules — filtered vs unfiltered (not open vs closed).
        -sVVersion detectionProbes open ports to determine service/version running.
        -sOIP protocol scanDetermines which IP protocols target supports.
        -sP / -snPing sweepHost discovery — finds live hosts without port scanning.
        OS & output flags
        SwitchFunction
        -OOS detection — fingerprints the operating system
        -AAggressive scan — enables -O, -sV, --traceroute, and script scanning (-sC)
        -p 80Scan specific port
        -p 1-1024Scan port range
        -p-Scan all 65535 ports
        -T0 to -T5Timing: T0=paranoid (IDS evasion), T3=default, T5=insane (fast, noisy)
        -oN fileNormal output to file
        -oX fileXML output
        -oG fileGrepable output
        -v / -vvVerbosity
        -D decoy1,decoy2Decoy scan — hides real IP among decoys
        --script <name>Run NSE script (e.g. --script smb-vuln-ms17-010)
        Port states
        StateMeaningResponse received
        OpenService is actively listeningSYN-ACK (TCP) or data (UDP)
        ClosedNo service listening, host reachableRST packet
        FilteredFirewall blocking — can't determine stateNo response or ICMP unreachable
        UnfilteredReachable but can't determine open/closedACK scan result
        Open|FilteredCan't distinguish — no responseUDP/FIN/Xmas/Null scan result
        Useful one-liners
        nmap -sS -sV -O -p- -T4 <target> nmap --script vuln <target> nmap -sS -p 1-1024 --open <target> nmap -sU -p 53,161,67 <target> nmap -sS -D RND:10 <target> nmap -sP 192.168.1.0/24
        Exam tip — Module 5
        The most tested Nmap concepts: -sS (SYN/stealth scan, default), -sT (TCP connect, logged), -sU (UDP, slow), -O (OS detection), -A (aggressive = everything). Know why SYN scan is "stealthy" and what each port state means. T0 = slowest/stealthiest, T5 = fastest/noisiest.
